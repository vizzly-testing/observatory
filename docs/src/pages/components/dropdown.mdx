# Dropdown

Dropdowns display a list of actions or options that appear when triggered. They're great for contextual menus and overflow actions.

## When to use

- **Action menus** — Edit, delete, duplicate actions
- **More options** — When space is limited for all buttons
- **Context menus** — Right-click style interactions
- **Overflow actions** — Secondary actions that don't fit

## Basic Usage

<ComponentPreview client:load>
  <DropdownBasicDemo />
</ComponentPreview>

```jsx
<Dropdown
  trigger={
    <Button variant="secondary" icon={EllipsisVerticalIcon}>
      Actions
    </Button>
  }
>
  <DropdownItem icon={PencilIcon} onClick={handleEdit}>Edit</DropdownItem>
  <DropdownItem icon={DocumentDuplicateIcon} onClick={handleDuplicate}>Duplicate</DropdownItem>
  <DropdownDivider />
  <DropdownItem icon={TrashIcon} variant="danger" onClick={handleDelete}>Delete</DropdownItem>
</Dropdown>
```

## Alignment

<ComponentPreview client:load>
  <DropdownPositionsDemo />
</ComponentPreview>

```jsx
<Dropdown trigger={...} align="left">...</Dropdown>
<Dropdown trigger={...} align="right">...</Dropdown>
```

## Item Variants

```jsx
// Normal item
<DropdownItem onClick={...}>Normal Action</DropdownItem>

// With icon
<DropdownItem icon={PencilIcon} onClick={...}>Edit</DropdownItem>

// Danger variant
<DropdownItem icon={TrashIcon} variant="danger" onClick={...}>Delete</DropdownItem>

// Disabled
<DropdownItem disabled>Unavailable</DropdownItem>

// Divider between groups
<DropdownDivider />
```

## Best Practices

<DosAndDonts client:load>
  <Do>Group related actions with dividers</Do>
  <Dont>Put too many items in one dropdown (max 7-8)</Dont>
</DosAndDonts>

<DosAndDonts client:load>
  <Do>Put destructive actions at the bottom with danger styling</Do>
  <Dont>Mix destructive actions randomly with other options</Dont>
</DosAndDonts>

<DosAndDonts client:load>
  <Do>Use icons consistently — all items or none</Do>
  <Dont>Add icons to only some items in the same dropdown</Dont>
</DosAndDonts>
