# Card

Cards group related content and actions. They create visual hierarchy and help users scan information.

## When to use

- **Content grouping** — Settings sections, build summaries, project cards
- **Data display** — Statistics, metrics, status overviews
- **Interactive containers** — Forms, action panels
- **List items** — When items need more structure than a table row


## Variants

<ComponentPreview client:load>
  <CardVariantsDemo />
</ComponentPreview>

| Variant | Usage |
|---------|-------|
| `default` | General content, no specific status |
| `success` | Approved builds, passing tests, positive outcomes |
| `warning` | Needs review, pending actions, attention required |
| `danger` | Failed builds, errors, critical issues |
| `info` | Informational, new items, processing |
| `purple` | Special categories, featured content |

## Best Practices

<DosAndDonts client:load>
  <Do>Use cards to group related content with clear visual boundaries</Do>
  <Dont>Nest cards inside cards — it creates confusing hierarchy</Dont>
</DosAndDonts>

<DosAndDonts client:load>
  <Do>Use variants consistently — success always means approved/passing</Do>
  <Dont>Use variants purely for decoration or to add visual variety</Dont>
</DosAndDonts>

<DosAndDonts client:load>
  <Do>Keep card content focused on a single topic or action</Do>
  <Dont>Cram unrelated information into one card</Dont>
</DosAndDonts>
