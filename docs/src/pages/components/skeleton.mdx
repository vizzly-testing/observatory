# Skeleton

Skeletons show placeholder content while data is loading. They reduce perceived wait time and prevent layout shifts.

## When to use

- **Initial load** — First time loading content
- **Data fetching** — While waiting for API responses
- **Image loading** — Placeholder while images load
- **Async content** — Any content that loads asynchronously

## Basic Usage

<ComponentPreview client:load>
  <SkeletonBasicDemo />
</ComponentPreview>

```jsx
<Skeleton variant="heading" />
<Skeleton variant="text" count={3} />
<div className="flex items-center gap-3">
  <Skeleton variant="avatar" />
  <div>
    <Skeleton variant="text" width="40%" />
    <Skeleton variant="text" width="60%" height="12px" />
  </div>
</div>
```

## Variants

<ComponentPreview client:load>
  <SkeletonVariantsDemo />
</ComponentPreview>

| Variant | Usage |
|---------|-------|
| `heading` | Page or section headings |
| `text` | Body text, descriptions |
| `avatar` | Circular user avatars |
| (custom) | Any size with `width` and `height` |

## Best Practices

<DosAndDonts client:load>
  <Do>Match skeleton layout to actual content layout</Do>
  <Dont>Use generic skeletons that don't match the real content</Dont>
</DosAndDonts>

<DosAndDonts client:load>
  <Do>Show skeletons for content that takes more than 300ms to load</Do>
  <Dont>Flash skeletons for very fast loads — it feels janky</Dont>
</DosAndDonts>

<DosAndDonts client:load>
  <Do>Animate skeletons to indicate loading is in progress</Do>
  <Dont>Use static gray boxes that look broken</Dont>
</DosAndDonts>
