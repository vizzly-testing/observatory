# Button

Buttons trigger actions. Choose the right variant based on the action's importance and consequences.

## When to use

- **Primary actions** — Submit forms, confirm dialogs, main CTAs
- **Secondary actions** — Alternative options, less prominent actions
- **Destructive actions** — Delete, remove, cancel subscriptions

## Variants

<ComponentPreview client:load>
  <ButtonVariantsDemo />
</ComponentPreview>

```jsx
<Button variant="primary">Primary</Button>
<Button variant="secondary">Secondary</Button>
<Button variant="ghost">Ghost</Button>
<Button variant="danger">Danger</Button>
<Button variant="warning">Warning</Button>
<Button variant="success">Success</Button>
```

| Variant | Usage |
|---------|-------|
| `primary` | Main action on the page. Use sparingly — typically one per view. |
| `secondary` | Alternative actions. Safe for multiple instances. |
| `ghost` | Tertiary actions, cancel buttons, navigation. Minimal visual weight. |
| `danger` | Destructive actions like delete. Always requires confirmation for irreversible actions. |
| `warning` | Caution-worthy actions that aren't destructive. |
| `success` | Positive confirmations, approvals. |

- **Large** — Hero sections, primary CTAs, mobile-friendly touch targets

## Best Practices

<DosAndDonts client:load>
  <Do>Use one primary button per view for the main action</Do>
  <Dont>Use multiple primary buttons that compete for attention</Dont>
</DosAndDonts>

<DosAndDonts client:load>
  <Do>Use danger variant for destructive actions</Do>
  <Dont>Use danger for emphasis or to draw attention to non-destructive actions</Dont>
</DosAndDonts>

<DosAndDonts client:load>
  <Do>Show loading state during async operations</Do>
  <Dont>Leave users wondering if their click registered</Dont>
</DosAndDonts>

<DosAndDonts client:load>
  <Do>Use concise, action-oriented labels: "Save", "Delete", "Export"</Do>
  <Dont>Use vague labels: "Submit", "OK", "Click Here"</Dont>
</DosAndDonts>
