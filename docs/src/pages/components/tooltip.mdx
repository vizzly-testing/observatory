# Tooltip

Tooltips display helpful text when users hover over or focus on an element. They provide additional context without cluttering the interface.

## When to use

- **Icon buttons** — Explain what icon-only buttons do
- **Truncated text** — Show full text on hover
- **Additional context** — Extra information about an element
- **Keyboard shortcuts** — Reveal shortcuts on hover

## Basic Usage

<ComponentPreview client:load>
  <TooltipBasicDemo />
</ComponentPreview>

```jsx

  <Button>Save Changes</Button>
</Tooltip>
```

## Best Practices

<DosAndDonts client:load>
  <Do>Keep tooltip text brief — aim for one short sentence</Do>
  <Dont>Put critical information only in tooltips — some users may never see them</Dont>
</DosAndDonts>

<DosAndDonts client:load>
  <Do>Use tooltips to explain icon-only buttons</Do>
  <Dont>Use tooltips on elements that already have clear labels</Dont>
</DosAndDonts>

<DosAndDonts client:load>
  <Do>Position tooltips to avoid covering important content</Do>
  <Dont>Position tooltips where they'll be cut off by viewport edges</Dont>
</DosAndDonts>

## Accessibility

- Tooltips appear on both hover and focus
- Content is announced by screen readers
- Avoid putting interactive elements inside tooltips
