# Avatar

Avatars represent users with profile images or generated initials. Use them to personalize interfaces and show user presence.

## When to use

- **User identification** — Profile pages, comments, activity feeds
- **Team displays** — Show project collaborators or reviewers
- **Attribution** — Who made changes, who approved builds
- **Presence indicators** — Online/offline status

## Basic Usage

<ComponentPreview client:load>
  <AvatarBasicDemo />
</ComponentPreview>

```jsx
<Avatar name="John Doe" />
<Avatar name="Jane Smith" src="/avatars/jane.jpg" />
```

## Sizes

<ComponentPreview client:load>
  <AvatarSizesDemo />
</ComponentPreview>

```jsx
<Avatar name="User" size="xs" />
<Avatar name="User" size="sm" />
<Avatar name="User" size="md" />
<Avatar name="User" size="lg" />
<Avatar name="User" size="xl" />
```

| Size | Pixels | Usage |
|------|--------|-------|
| `xs` | 24px | Compact lists, inline mentions |
| `sm` | 32px | Table cells, tight spaces |
| `md` | 40px | Default, most contexts |
| `lg` | 48px | Emphasized displays |
| `xl` | 64px | Profile headers, hero sections |

## Avatar Groups

Use `AvatarGroup` to display multiple avatars with overflow handling:

<ComponentPreview client:load>
  <AvatarGroupDemo />
</ComponentPreview>

```jsx
<AvatarGroup
  avatars={[
    { name: 'Alice', src: '/avatars/alice.jpg' },
    { name: 'Bob', src: null },
    { name: 'Carol', src: '/avatars/carol.jpg' },
  ]}
  max={3}
/>
```

## Best Practices

<DosAndDonts client:load>
  <Do>Always provide a name for accessibility and fallback initials</Do>
  <Dont>Use avatars without any identifying information</Dont>
</DosAndDonts>

<DosAndDonts client:load>
  <Do>Use consistent sizing within the same context</Do>
  <Dont>Mix different avatar sizes in the same row or list</Dont>
</DosAndDonts>
