# Table

Tables display structured data in rows and columns. Use for content that benefits from direct comparison.

## When to use

- **Structured data** — Lists with multiple attributes per item
- **Comparison** — When users need to compare values across rows
- **Scannable content** — Data that users scan rather than read
- **Simple lists** — When DataTable features aren't needed

## Basic Usage

<ComponentPreview client:load>
  <TableBasicDemo />
</ComponentPreview>

```jsx
let columns = [
  { key: 'name', header: 'Screenshot' },
  {
    key: 'status',
    header: 'Status',
    render: (value) => <Badge variant={...}>{value}</Badge>,
  },
  { key: 'changes', header: 'Changes' },
  { key: 'updated', header: 'Updated' },
];

<Table
  columns={columns}
  data={data}
  onRowClick={(row) => navigate(`/screenshot/${row.id}`)}
/>
```

## Table vs DataTable

| Use Table when... | Use DataTable when... |
|-------------------|----------------------|
| Simple display | Sorting needed |
| No user configuration | Column visibility toggle |
| Few rows | Many rows with filtering |
| Read-only | Interactive features |

## Custom Rendering

Use the `render` function in column definitions to customize how cells display:

```jsx
{
  key: 'status',
  header: 'Status',
  render: (value, row) => (
    <Badge variant={value === 'approved' ? 'success' : 'warning'}>
      {value}
    </Badge>
  ),
}
```
