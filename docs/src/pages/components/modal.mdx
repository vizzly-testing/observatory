# Modal

Modals display focused content that requires user attention. They overlay the page and block interaction with the background until dismissed.

## When to use

- **Confirmations** — Important decisions before proceeding
- **Forms** — Creating or editing content
- **Details** — Viewing more information without leaving context
- **Focused tasks** — Multi-step wizards, settings

```jsx
<Modal
  isOpen={isOpen}
  onClose={() => setIsOpen(false)}
  title="Update Baseline"
  description="This will update your baseline images"
>
  <p>Modal content here...</p>
  <div className="flex justify-end gap-3">
    <Button variant="ghost" onClick={close}>Cancel</Button>
    <Button onClick={handleConfirm}>Confirm</Button>
  </div>
</Modal>
```

## Best Practices

<DosAndDonts client:load>
  <Do>Use clear titles that describe the modal's purpose</Do>
  <Dont>Use vague titles like "Confirm" or "Are you sure?"</Dont>
</DosAndDonts>

<DosAndDonts client:load>
  <Do>Provide a clear way to close (X button, Cancel, click outside)</Do>
  <Dont>Trap users in modals without an exit</Dont>
</DosAndDonts>

<DosAndDonts client:load>
  <Do>Keep modal content focused on a single task</Do>
  <Dont>Nest modals inside modals</Dont>
</DosAndDonts>

## Accessibility

- Focus is trapped inside the modal while open
- Pressing Escape closes the modal
- Focus returns to the trigger element when closed
- Background content is hidden from screen readers
