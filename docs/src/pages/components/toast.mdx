# Toast

Toasts display brief, temporary notifications. They appear, deliver a message, and automatically disappear.

## When to use

- **Background operations** — "Changes saved", "Uploaded successfully"
- **Non-critical feedback** — Confirmations that don't require action
- **Async completions** — When background tasks finish
- **Transient status** — Brief updates that don't need permanent visibility

## Interactive Demo

Click the buttons to trigger different toast variants:

  <ToastDemo />

```jsx
<Toast
  isVisible={true}
  message="Changes saved successfully!"
  variant="success"
  onClose={() => setVisible(false)}
  position="top-right"
/>
```

## Variants

| Variant | Usage |
|---------|-------|
| `success` | Completed actions, saved changes |
| `warning` | Attention needed, potential issues |
| `error` | Failed operations, errors |
| `info` | Neutral notifications |

## Positions

Use `top-right` for most cases. Consider `bottom-*` positions for mobile or when the top of the screen has important content.

## Best Practices

<DosAndDonts client:load>
  <Do>Keep toast messages brief — aim for under 10 words</Do>
  <Dont>Use toasts for critical information that requires user action</Dont>
</DosAndDonts>

<DosAndDonts client:load>
  <Do>Use consistent positions throughout your application</Do>
  <Dont>Show multiple toasts stacked at once — queue them or consolidate</Dont>
</DosAndDonts>

## Toast vs Alert

| Use Toast when... | Use Alert when... |
|-------------------|-------------------|
| Message is transient | Message should persist |
| No action required | User needs to take action |
| Confirming completed action | Showing validation errors |
| Non-blocking feedback | Critical information |
