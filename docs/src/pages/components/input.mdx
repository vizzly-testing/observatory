# Input

Text inputs allow users to enter and edit text. They're the foundation of form-based interactions.

## When to use

- **Text entry** — Names, emails, URLs, search queries
- **Short answers** — Single-line responses
- **Formatted data** — Passwords, phone numbers, codes
- **Search fields** — Quick filtering and lookup

## Basic Usage

<ComponentPreview client:load>
  <InputBasicDemo />
</ComponentPreview>

```jsx

```

## Best Practices

<DosAndDonts client:load>
  <Do>Always include a label — it's essential for accessibility</Do>
  <Dont>Use placeholder text as the only label</Dont>
</DosAndDonts>

<DosAndDonts client:load>
  <Do>Use appropriate input types (email, password, url, etc.)</Do>
  <Dont>Use a generic text input when a specific type would help</Dont>
</DosAndDonts>

<DosAndDonts client:load>
  <Do>Show validation errors inline, near the input</Do>
  <Dont>Clear the input when showing an error — let users fix it</Dont>
</DosAndDonts>
