# Toggle

Toggles provide an immediate on/off switch. Unlike checkboxes, toggles apply changes immediately without needing to submit a form.

## When to use

- **Immediate effect** — Changes that apply instantly
- **Feature toggles** — Enable/disable functionality
- **Settings** — Dark mode, notifications, auto-save
- **Visibility** — Show/hide elements

## Basic Usage

<ComponentPreview client:load>
  <ToggleBasicDemo />
</ComponentPreview>

```jsx
<Toggle
  label="Dynamic content detection"
  description="Auto-detect and ignore dynamic regions"
  checked={enabled}
  onChange={(e) => setEnabled(e.target.checked)}
/>
```

## Best Practices

<DosAndDonts client:load>
  <Do>Use toggles for settings that take effect immediately</Do>
  <Dont>Use toggles in forms where changes need to be submitted</Dont>
</DosAndDonts>

<DosAndDonts client:load>
  <Do>Place the toggle on the right side of its label</Do>
  <Dont>Place toggles in unpredictable positions</Dont>
</DosAndDonts>

<DosAndDonts client:load>
  <Do>Show visual feedback when the toggle state changes</Do>
  <Dont>Leave users unsure if their action was registered</Dont>
</DosAndDonts>
